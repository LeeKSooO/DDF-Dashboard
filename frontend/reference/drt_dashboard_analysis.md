# DRT 대시보드 개발 회의록 종합 분석

## 1. 프로젝트 개요
- **목적**: DRT(Demand Responsive Transport) 서비스를 위한 교통 데이터 분석 대시보드 개발
- **범위**: 서울시 버스 승하차 데이터 기반 시각화 및 정책 분석
- **팀 구성**: 발화자1(지도교수/컨설턴트), 발화자2(데이터 분석가), 발화자3(프론트엔드 개발자), 발화자4(기타 팀원)

## 2. 대시보드 구조 및 주요 기능

### 2.1 메인 대시보드 구성요소
**상단 고정 영역**
- 서울시 전체 교통량 현황
- 구별 총 정류장 수 및 평균 이용량
- 교통 집중도 지표

**동적 업데이트 영역 (구 선택 시 변경)**
- 인기 정류장 TOP 5
- 구별/정류장별 상세 정보
- 교통량 히트맵

### 2.2 핵심 탭 구성
1. **개요(Overview)**: 전체 교통 현황 대시보드
2. **교통 분석**: 시간대별/요일별 패턴 분석
3. **교통 히트맵**: 지역별 교통량 시각화
4. **DRT 분석**: 적합도 모델 결과

## 3. 주요 개선사항 및 피드백

### 3.1 UI/UX 개선
**레이아웃 최적화**
- 고정값과 동적값의 명확한 구분
- 공간 활용도 개선 (현재 너무 넓게 사용)
- 카드 형태의 컴포넌트로 정보 압축

**인터랙션 개선**
- 구 선택 시 관련 데이터 연동 업데이트
- 애니메이션 효과를 통한 부드러운 전환
- 상세보기 팝업 기능 추가

### 3.2 지도 시각화 강화
**정류장 표시 방식**
- 상위 5개 정류장을 지도에 색상으로 강조
- 크기와 색상을 함께 활용한 이중 인코딩 검토
- 범례 값 조정으로 색상 다양성 확보

**지리적 맥락 제공**
- 강남구-서초구 경계선의 정류장 밀집 현상 설명
- 지역 특성과 교통 패턴의 연관성 시각화

### 3.3 데이터 표현 개선
**용어 정확성**
- "승객수" → "승하차수"로 수정
- "해당 구 평균 대비" → "평균 대비"로 간소화
- 배율 표시 시 "X" 크기 조정

**지표 추가 및 개선**
- 동별 평균 정류장 수 추가
- 정류장당 이용량 평균값 명시
- 교통 불평등도 지표 도입

## 4. 핵심 인사이트 및 분석 결과

### 4.1 시간대별 교통 패턴 분석
**새벽 시간대 특성**
- 도봉구, 강북구, 은평구: 새벽 4-7시 높은 이용률
- 원인: 건물 청소, 도매시장 관련 직업군의 출근 패턴
- DRT 필요성: 첫차 대기 수요 및 교통약자 지원

**병원가 오후 피크**
- 강남구, 서초구: 오후 2-3시 특별한 피크 발생
- 원인: 수술 입원(아침 식사 전), 퇴원(점심 이후) 시간대
- 대중교통 배차간격 미반영으로 인한 DRT 수요

**심야 고수요 지역**
- 강남역, 수서역 등: 23시-새벽 3시 높은 수요
- 회식, 야근 후 귀가 수요와 심야버스 부족 문제

### 4.2 지역별 특성 분석
**동대문구 사례**
- 청량리역: 지방 연결 교통허브 + 경동시장(한약재) 복합 특성
- 새벽 고수요: 도매시장 특성 반영
- 주말 우세: 관광/쇼핑 목적 이용객

**강남구 특성**
- 대치/한티역: 학원가 특성으로 인한 점심시간 고수요
- 래미안아파트 지역: 주거지역이지만 높은 승차율 의문점

### 4.3 교통 불평등도 개념 도입
**정의 및 의미**
- 구 내 정류장별 이용량의 격차를 나타내는 새로운 지표
- 높은 불평등도 = DRT 도입 효과가 클 것으로 예상되는 지역
- 기존 연구나 정책에서 다루지 않은 독창적 개념

**활용 방안**
- 향후 인구 데이터(동별 주민 수) 결합 시 더욱 정확한 분석 가능
- 정류장 수 대비 주민 수 비율을 통한 교통 형평성 분석

## 5. 정책적 시사점 및 문제점 발견

### 5.1 현행 DRT 정책의 한계
**레드존-그린존 정책의 경직성**
- 현재: 레드존↔레드존 간 이동 금지, 시간대 무관
- 문제점: 점심시간 레드존 내 높은 수요에도 서비스 제공 불가
- 개선방향: 시간대별 레드존-그린존 구분 필요

**구체적 사례**
- 점심시간 강남 지역: 5000-8000명 수요이지만 레드존 간 이동 제한
- 심야시간 강남역 주변: 레드존이지만 그린존 역할 필요

### 5.2 정책 개선 제안
**시간대별 존 구분 도입**
- 출퇴근 시간: 기존 레드존-그린존 적용
- 점심/심야 시간: 유연한 존 운영 필요
- 지역별 특성을 반영한 차별화된 정책

## 6. 데이터 품질 및 검증 이슈

### 6.1 데이터 전처리 필요사항
**정류장 명칭 통합**
- 동일 지역 복수 정류장의 합산 처리
- 정류장 ID 활용한 정확한 식별
- 중복 표시 문제 해결

**이상치 처리**
- 서울의료원 후문 같은 극단적 저이용 정류장 처리
- 지리적 위치와 실제 접근성 고려한 필터링

### 6.2 승하차 데이터 분석
**승차 vs 하차 분리 분석의 중요성**
- 현재: 승하차 통합 데이터 사용
- 개선: 승차/하차 분리를 통한 지역 특성 파악
- 예: 주거지역(승차 많음) vs 업무지역(하차 많음) 구분

- 데이터 업데이트 주기 및 지연 시간 명시

- 종합 교통 분석 플랫폼으로 발전

## 10. 결론 및 의의

### 10.1 프로젝트의 성과
**기술적 성과**
- 복잡한 교통 데이터의 직관적 시각화 구현
- 인터랙티브 대시보드를 통한 탐색적 분석 가능
- 다양한 시간축과 공간축 분석의 통합 제공

**정책적 기여**
- 기존 DRT 정책의 한계점 데이터 기반 발견
- 교통 불평등도라는 새로운 지표 제안
- 지역별/시간별 특성을 반영한 세분화된 정책 방향 제시

### 10.2 한계점 및 향후 과제
**데이터 한계**
- 단일 시점(7월) 데이터로 인한 계절성 미반영
- 버스 데이터만으로 한정된 교통 현황 분석
- 실제 DRT 수요와 대중교통 이용 패턴 간 차이 고려 필요

**정책 적용의 현실적 제약**
- 기존 교통 인프라와의 연계 방안
- 운영비용 대비 효과성 검증 필요
- 이해관계자 간 조정 및 합의 과정

이 프로젝트는 단순한 데이터 시각화를 넘어서 교통 정책의 실질적 개선 방향을 제시하는 의미있는 성과를 달성했다고 평가됩니다.
